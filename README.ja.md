# テキスト型 PDF → 注釈付き HTML ビューア

> PDF が**テキスト型**（選択可能な文字が埋め込まれている、単なるスキャン画像ではない）であれば、
> **文字を選択・ハイライト・注釈できる Web ページ**に変換できます —— 完全ローカル、外部サービス不要、CDN 不要。

教科書 / 論文 / ドキュメントを放り込むと、PDF と同じように**ページ上で文字を選択**し、ワンクリックでハイライトや下線を引き、メモを書けます。注釈は更新しても消えず、**自己完結型の HTML ファイルとして書き出し**（単一ファイルなので他人に送っても注釈を続けられます）。

---

## ✨ 解決する課題

| 従来の方法 | 本ツール |
|------------|----------|
| スクショ OCR、PNG だらけでファイル数爆発 | **テキスト＋座標**のみ抽出。1 冊 ≈ 2 つの JSON でディスク負荷ほぼゼロ |
| 注釈が PDF リーダー内にロックされる | 注釈は Web ページに保存され、書き出し・共有・編集可能 |
| クラウドストレージ / オンライン SaaS に依存しプライバシー漏洩 | 完全ローカル。元 PDF は抽出直後に削除 |
| 文字が選択できずコピーが面倒 | 文字はネイティブに選択・コピー可能 |

> 💡 **設計思想**: パソコンにゴミを溜めず、ファイル数を増やさない。元 PDF はアップロード後すぐに抽出して破棄し、ディスクには極小の構造化データだけを残す。

---

## 🚀 クイックスタート

```bash
# 1) 依存関係をインストール（たった 1 つ）
pip install -r requirements.txt

# 2) 起動（純標準ライブラリ、Flask / データベース不要）
python app.py
# 既定 http://127.0.0.1:8000
# 同一 LAN で共有する場合: python app.py --host 0.0.0.0 --port 9000

# 3) ブラウザを開き、テキスト型 PDF をアップロードして注釈を始める
```

Python 3.8 以上が必要です。`pymupdf` 以外に実行時依存はありません。

---

## 🖱 使い方

1. **アップロード**: ホーム画面で任意のテキスト型 PDF をドラッグ / 選択。
2. **閲覧 ＋ 注釈**:
   - **マウスで文章を選択**するとツールバーが表示 → 「ハイライト / 下線 / 注釈」を選択。
   - 「注釈」で入力欄が開き、メモを記入。注釈は右サイドバーに表示され、いつでも編集・削除可能。
   - 上部のスライダーでページを拡大縮小（0.5×–3×）。長文書は自動で遅延レンダリングし、カクつかない。
3. **注釈の永続化**: オンライン版の注釈はサーバーの `data/` に保存され、更新やブラウザ再起動後も消えない。
4. **書き出し**: 「Export HTML」をクリック → 全テキスト・レイアウト・注釈を含む**単一の `.html` ファイル**を取得。
   ダブルクリックで開き、**オフラインで注釈可能**。そのまま同学・同僚に送って注釈を続けられる。

> ⚠️ 対応は**テキスト型** PDF のみ。純スキャン（埋め込み文字なし）は「抽出可能な文字がありません」と表示されます。
> このようなファイルは先に OCR ツールでテキスト PDF に変換してからアップロードしてください。

## 📂 プロジェクト構成

```
.
├── app.py            # バックエンド: 純標準ライブラリ HTTP サービス（アップロード / 読取 / 注釈 / 書出）
├── pdftext.py        # PDF テキスト＋レイアウト抽出（PyMuPDF）、コンパクトな JSON を出力
├── static/
│   ├── index.html    # アップロード画面 ＋ 閲覧・注釈画面（デュアルビュー、同一 JS）
│   ├── style.css     # スタイル
│   └── app.js        # フロントエンド処理: レンダリング / 選択→注釈マッピング / サイドバー / 書出
├── requirements.txt  # pymupdf のみ
├── poster.pdf        # 小さなサンプル（デモ用、追跡対象）
└── data/             # 実行時に生成: 文書ごとの抽出結果 ＋ 注釈（gitignore、リポジトリに含まれない）
```

## 🔌 API（HTTP）

| メソッド | パス | 説明 |
|----------|------|------|
| GET  | `/api/info` | サービス情報 |
| GET  | `/` または `/index.html` | フロントエンド画面 |
| POST | `/api/upload` | 生 PDF バイト ＋ ヘッダ `X-Filename`；抽出後 `{id, title, page_count}` を返す |
| GET  | `/api/doc/<id>` | 抽出結果（テキスト span / 画像 / メタ情報） |
| GET  | `/api/docs` | アップロード済み文書の一覧 |
| GET/POST | `/api/annotations/<id>` | 注釈の読取 / 上書き保存（JSON 配列） |
| GET  | `/api/export/<id>` | **自己完結型の注釈付き HTML** を書き出し（単一ファイル） |
| DELETE | `/api/doc/<id>` | 文書とその注釈を削除 |

> 書き出したスタンドアロン HTML は注釈を `localStorage` に保存し、バックエンド不要で動作します。

---

## 🧱 技術スタック

- **バックエンド**: Python 標準ライブラリ `http.server`（スレッドセーフ）、フレームワーク・データベース不要。
- **抽出**: PyMuPDF 1.24+（`fitz`）、span ごとに文字・フォントサイズ・太さ・色・座標を抽出。
- **フロントエンド**: バニラ HTML / CSS / JS、ビルド不要、npm 依存なし。
- **保存**: 文書ごとに `data/<id>/` ディレクトリを作成し `doc.json` と `annotations.json` を格納。
